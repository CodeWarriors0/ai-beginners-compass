第三阶段：理解 AI 的三种形态

学习时长：1 天 · 学习目标：分清"聊天 AI / CLI AI / Agent"，知道什么场景该用哪种

到了第二阶段你已经在用"聊天 AI"了。但 AI 不只这一种形态——还有能进入你电脑动手干活的、能自己拆任务连续执行的。这一阶段我们把市面上常听到的"AI 产品"理清楚，让你以后看到任何新工具都能一眼归类。

本节不需要动手安装，看懂概念即可。具体安装与使用，留到课外教程和后续进阶部分。

一、为什么要分清楚形态？

很多人把"AI"当成一个东西，结果遇到问题搞混：

  • 看到博主说"AI 能帮我配置路由器" → 想当然以为豆包能做，结果发现根本不行
  • 听说"Agent 智能体很强大" → 不知道指的是一个产品还是一种能力
  • 纠结"我到底该用 ChatGPT 还是 Claude Code 还是扣子"

其实它们处于不同的"形态层级"，能力和适用场景完全不同。下面这张图你按场景挑形态，以后再看到新工具，也能秒判断：

二、形态一：聊天 AI（你已经熟悉）

💬 聊天 AI（Chat AI）

代表产品：豆包、Kimi、文心一言、通义千问、ChatGPT、Claude

怎么工作：你在它的网页或 App 输入框打字，它回答你。它住在云端（厂商的服务器上），通过网络给你回话。

能力边界：

  • ✅ 聊天、回答问题、生成文字/图片/视频
  • ✅ 读你上传的文件、看你发的图
  • ✅ 联网查最新信息（开了联网功能的）
  • ❌ 不能看到你电脑里其他文件
  • ❌ 不能在你电脑上执行任何动作
  • ❌ 不能访问你的微信、邮箱等私人数据
典型场景：写文案、查信息、解释概念、翻译、做表格——也就是前两阶段你已经在用的所有事。
聊天 AI 就像你加了一个万事通的微信好友。你发消息他回消息，仅此而已。他看不见你家里的东西，也不会替你去做事。

三、形态二：CLI AI（命令行 AI）

🖥 CLI AI（命令行 AI）

代表产品：Claude CodeAnthropic 出的命令行 AI 工具，运行在你电脑的终端里，能读你的文件、跑命令、改代码。底层调用 Claude 大模型。、Codex CLIOpenAI 出的命令行 AI 工具，类似 Claude Code，底层调用 GPT 系列模型。、Gemini CLI、Aider、Cursor、Windsurf

怎么工作：你把它装到自己电脑上（通常通过命令行英文叫 Terminal 或 CLI（Command Line Interface），是一种用打字命令而不是点按钮来操作电脑的方式。Mac 上叫"终端"，Windows 上叫"PowerShell"或"命令提示符"。启动），它就能：

  • ✅ 看你当前文件夹里的所有文件
  • ✅ 真的去执行命令（如改文件、装软件、连服务器）
  • ✅ 连续多步完成一个任务（出错自己换法子重试）
  • ⚠️ 每个"动手"动作默认会问你"允许吗？"

它和聊天 AI 的关键区别：它不只是"说"，而是"做"。

典型场景：修改代码、批量重命名文件、读写 Excel / CSV 表格（汇总、清洗、跨表合并）、诊断网络问题、SSH 登录路由器改配置、把照片批量压缩——一切"需要动你电脑"的事。
CLI AI 就像一个上门服务的师傅。你把家门钥匙给他，他能进屋、检查情况、亲手修东西。强大，但也必须当面盯着，不能放任他乱来。
能力越大、风险越大。CLI AI 默认每次"动手"前会请示，不要无脑放行，否则可能误删文件、改坏配置。

四、形态三：Agent / 智能体

🤖 Agent / 智能体

代表平台：扣子（Coze）、豆包智能体、Kimi+ 应用、GPTs、Manus、AutoGPT

Agent 不是一个具体产品，而是一种"能力描述"——指能自己拆任务、调用工具、连续执行多步的 AI 程序。

表现形式：

  • 可以是云端的（扣子上做的智能体、GPTs、Manus）
  • 也可以是本地的（CLI AI 本身就具有 Agent 能力）
  • 可以是文字界面，也可以是浏览器自动操作

核心特征 —— 自主性：

  • 给它一个目标（"帮我订下周三北京到上海的高铁，要二等座靠窗"）
  • 它自己拆步骤：查车次 → 选座 → 下单 → 支付确认
  • 遇到障碍自己想办法（比如靠窗没了，问你"换过道行不行"）
典型场景：把一段你常用的"提示词 + 知识库 + 工具"封装成专用机器人，下次直接调用，不用重复交代。比如"专门给娃讲题的解题助手"、"翻译官"、"简历优化师"。
Agent 像"能自己干活的 AI"这个职业类别。微信好友（聊天 AI）可以是 Agent，上门师傅（CLI AI）也可以是 Agent——它描述的是"能不能自己完成多步任务"，不是某一个具体产品。

五、形态四：IDE 插件 AI（嵌进编辑器里的助手）

🧩 IDE 插件 AI（IDE Plugin AI）

代表产品：CC GUI（IntelliJ / PyCharm 等 JetBrains 全家桶）、GitHub Copilot、Claude Code for VS Code、Cursor / Windsurf（独立 IDE）、通义灵码、豆包 MarsCode

怎么工作：把 AI 直接嵌进你写代码的编辑器里。AI 能看到你当前打开的文件、光标位置、整个工程结构，边敲代码边帮你写，也能在侧边栏聊天。

它和 CLI AI 的区别：底层模型/能力相近，但界面在 IDE 里——可以图形化点选改动、查看 diff、点跳转、看高亮——比纯命令行直观得多，对程序员更友好。

能力边界：

  • ✅ 看到整个工程的代码上下文（@ 引用文件、# 引用 agent）
  • ✅ 把 AI 改动以 diff 形式展示，逐处确认是否采纳
  • ✅ 跑命令、查文件（继承了 CLI AI 的能力）
  • ✅ 通过 MCP 扩展更多工具
  • ⚠️ 主要面向编程场景，非程序员日常用不上
典型场景：写代码时让 AI 自动补全整段函数、解释陌生代码、修 bug 时让它直接改、跨文件重构、把 AI 给的方案以 diff 形式 review 后再应用。
IDE 插件 AI 就像给上门师傅配了一个透明工作台——他动你电脑的同时，每一步改动都在你面前的屏幕上画给你看（红色减、绿色加），你点一下就接受或拒绝。同样的能力，体验比纯命令行(CLI AI) 友好得多。
比如 JetBrains 系（IDEA / PyCharm / WebStorm 等）的同学，可以装 CC GUI 插件，直接在 IDE 里用 Claude Code / Codex 的全部能力——支持双引擎切换、@ 引用文件、可视化 diff、MCP 集成，免去开终端。

八、新手最容易搞错的 4 件事

❌ 误解 1：以为 ChatGPT / 豆包能进我电脑帮我整理文件。
✅ 真相：聊天 AI 没有进入你电脑的权限。要进电脑只能用 CLI AI 或 IDE 插件 AI。

❌ 误解 2：以为"Agent 智能体"是一种新产品。
✅ 真相：它是一种能力。豆包智能体、扣子智能体、GPTs 都是不同平台对这种能力的实现。

❌ 误解 3：以为 CLI AI 比聊天 AI 更"聪明"。
✅ 真相：它们背后用的可能是同一个大模型。区别在权限和工具，不在智力。

❌ 误解 4：以为新手就该直接上 CLI AI 或 Agent。
✅ 真相：聊天 AI 已经能覆盖 80% 的日常需求。先把聊天 AI 用熟，再按需进阶。
