cc-switch 安装与使用教程课外 · 国内模型一键切换

让 Claude Code / Codex / Gemini CLI 一键切到 DeepSeek / Kimi / 智谱 / 通义 / 豆包等国内模型

用过 Claude Code / Codex / Gemini CLI 的同学都会发现一个痛点：它们默认连的是海外厂商，国内网络不稳，注册/付费门槛也高。

cc-switch由 farion1231 开源的桌面 App（Tauri + React），把 50+ 个国内外 API 服务商做成预设，让你在 Claude Code / Codex / Gemini CLI / OpenCode / OpenClaw 之间一键切换"用哪家的模型"。 就是来解决这个的：一个图形界面，50+ 个预设服务商，点一下就能把这些 CLI 切换到 DeepSeek、Kimi、智谱 GLM、通义千问、豆包等国内模型（很多还有免费额度）。

一、它到底解决了什么问题？

把 Claude Code 想成一台"咖啡机"——默认装的是 Anthropic 牌咖啡豆（要外汇买、运得慢）。cc-switch 就像一个豆子转换器：插上后，你可以一键换成猫屎咖啡（Claude）、瑞幸（DeepSeek）、星巴克（Kimi）……机器还是那台，只是豆子换了。
用 cc-switch 前用 cc-switch 后
手动改 ~/.claude/settings.json 里的 API 地址和 key 图形界面点一下即可切换
记不住每家服务商的 base URL 和模型名 50+ 预设，含字段全自动填好
切换还要重启终端 Claude Code 切换无需重启，其它工具重开终端即可
不同 CLI 工具的配置分散在各处 统一管理 Claude Code / Codex / Gemini CLI / OpenCode / OpenClaw

二、安装

macOS 推荐用 Homebrew

如果你已经装了 Homebrew，复制下面两行到终端：

$ brew install --cask cc-switch # 以后想升级 $ brew upgrade --cask cc-switch

没装 Homebrew？也可以从 GitHub Releases 下 .dmg 文件，双击安装。

Windows

去 GitHub Releases 下载：

  • CC-Switch-v{版本号}-Windows.msi —— 标准安装包（推荐）
  • CC-Switch-v{版本号}-Windows-Portable.zip —— 绿色版（解压即用，不写注册表）

Linux

Arch / Manjaro 用户：

$ paru -S cc-switch-bin

其它发行版从 Releases 下 .deb / .rpm / .AppImage 都行。

三、第一次使用（4 步上手）

1打开 cc-switch
从应用程序里启动，会看到主界面（如上图）。左侧是"应用"列表（Claude Code / Codex / Gemini CLI 等），中间是当前应用的"供应商"列表。
2添加一个供应商
点右上角"添加"，会弹出预设列表（DeepSeek、Kimi、智谱 GLM、通义、豆包、SiliconFlow、AnyRouter……）。挑一个有免费额度的试水，比如 SiliconFlow 或 DeepSeek 都不错。
图 2 ｜ 添加供应商：选预设 + 填 API Key（来源：项目官方 GitHub README）
3填 API Key
选定预设后，会让你填 API Key。这个去对应服务商网站注册即可（多数支持微信/手机号登录），首次注册一般送几块到几十块的免费额度。base URL、模型名等字段已经自动填好，不用动。
4点"启用"
添加好后，在供应商卡片上点"启用"。Claude Code 当前会话立即生效；Codex / Gemini CLI 需要关掉再重开终端。

切换好之后正常打开 Claude Code：

$ claude # 此时它已经在用国内模型回话，不再走 Anthropic 海外服务器

四、常见国内服务商一览（cc-switch 已预设）

服务商能用的模型特点
DeepSeek 官方DeepSeek-V3、R1价格便宜、推理强、注册即用
Kimi（月之暗面）Kimi-K2、Moonshot 系列长上下文擅长、国内体验流畅
智谱 BigModelGLM-4.6、GLM-4.5有 Coding 套餐、写代码效果好
通义（阿里）Qwen-Max、Qwen-Coder免费额度大、模型迭代快
豆包（字节）Doubao 系列对中文友好、客服活跃
SiliconFlow（硅基流动）聚合多家国产模型一个 key 通吃多家、有免费额度
AnyRouter / OpenRouter 等中转声称支持 Claude / GPT 系方便但需要看清是否官方授权
新手第一次玩，推荐路径：SiliconFlow → 注册 → 拿 API Key → 在 cc-switch 里选 SiliconFlow 预设填 Key → 启用。一条龙不到 5 分钟。

五、切回 Claude / Codex 官方

哪天你想切回 Anthropic 官方账号：

  1. 在 cc-switch 里添加预设 "Anthropic Official Login"
  2. 点"启用"
  3. 重启你的 Claude Code 终端
  4. 按提示走一遍 OAuth 登录就回到了官方

七、5 个常见踩坑

❌ 坑 1：切换后报 401 / 403 错误
✅ 多半是 API Key 填错或额度耗尽。先到对应服务商网站登录看看额度，再回 cc-switch 编辑这个供应商重填 Key。

❌ 坑 2：切换后报"model not found"
✅ 服务商支持的模型名可能更新过。在供应商详情里把模型名换成它官网当前提供的（比如 deepseek-chat → deepseek-v3）。

❌ 坑 3：Codex / Gemini CLI 切完没反应
✅ Codex / Gemini CLI 都需要关掉再重开终端才能读到新配置。只有 Claude Code 是热切换。

❌ 坑 4：用中转 / 转发服务遇到限速或封号
✅ 第三方中转可能违反原厂 ToS，账号有被封的风险。正式工作场景请直接用国产模型官方 API。

❌ 坑 5：找不到配置文件想手动改
✅ cc-switch 的数据放在 ~/.cc-switch/cc-switch.db（SQLite）。Claude Code 真正读的还是 ~/.claude/settings.json，cc-switch 会自动同步它。

安全提醒：API Key 就是钱。不要把含 Key 的配置文件发到群里、传到 GitHub 公共仓库、贴到 AI 对话框。cc-switch 本地存储是加密的，但导出/截图时要注意打码。

☐ 我知道 cc-switch 是干什么的（一句话讲清楚）
☐ 我按系统装好了 cc-switch，能打开主界面
☐ 我至少添加并启用了 1 个国内模型供应商

☐ 我知道切换 Codex / Gemini CLI 后要重开终端
☐ 我清楚 API Key 不能外泄
小练习：去 SiliconFlow 或 DeepSeek 开放平台 注册，拿到 API Key 后在 cc-switch 里启用，然后在终端跑 claude 并问它"你是谁？背后用的什么模型？"——验证切换是否真的生效。
