课外教程：本地 AI 命令行工具课外 · 进阶

认识 Claude Code、Codex 这类"住在你电脑里"的 AI，以及它们能干什么对话 AI 干不了的事

本节是给已经走完正课、想再进一步的同学。在第三阶段你已经了解了"聊天 AI / CLI AI / Agent"三种形态的区别——本节聚焦其中的 CLI AI：怎么装、怎么用，以及它能在家庭场景里帮你做哪些实在事（优化网络、配置软路由、批处理文件等）。

一、几个主流的 CLI AI 工具

工具 归属 底层模型 用什么装 费用
Claude Code Anthropic Claude npm install -g @anthropic-ai/claude-code 订阅 Claude Pro 或 API 计费
Codex CLI OpenAI GPT npm install -g @openai/codex OpenAI API key 或 ChatGPT Plus
Gemini CLI Google Gemini npm install -g @google/gemini-cli 免费额度 + 付费
Aider 开源 可接任意模型 pip install aider-chat 取决于你接的模型
Cursor / Windsurf 独立公司 多模型 下载图形界面 App 免费 + 订阅
使用前提（每一项独立一行，逐条核对）：
  1. 一台 Mac / Linux / Windows 电脑
  2. 会打开"终端"（macOS：Terminal；Windows：PowerShell）
  3. 有 Node.js / npm（点上面表格里的命令跳转 npm 安装教程）或 Python 环境
  4. 能稳定访问海外网络（Claude / OpenAI / Gemini 的官方服务在国外）
  5. 替代方案：用 cc-switch 一键把 Claude Code / Codex / Gemini CLI 切到国内模型（DeepSeek、Kimi、智谱 GLM、通义、豆包等），就不再依赖海外网络
  6. 国内原生 CLI 工具：通义灵码 CLI、豆包 MarsCode、智谱 Coding（多偏编程场景）

二、安装与第一次使用（以 Claude Code 为例）

下面用 Claude Code 做一个最短上手演示，其它工具流程类似。

1. 准备环境

需要先装 Node.js（去 nodejs.org 下 LTS 版）。装完打开终端验证：

$ node -v
v22.10.0

2. 安装

$ npm install -g @anthropic-ai/claude-code

3. 启动并登录

$ claude
# 首次会让你登录 Anthropic 账号

4. 给它下第一个指令

> 你能看到我这个目录吗？列一下里面有什么文件，并告诉我每个大概是干什么的。

它会调用 ls、cat 等命令，把目录内容整理成易读的中文摘要给你。

装完后，先在一个空目录里试用，不要直接对着重要文件下命令。每个操作请求都看清再放行。

三、CLI AI 的三块积木：Tools / MCP / Skills

你装完 Claude Code 后会发现它"突然就能干很多事"——读文件、跑命令、连服务器、改配置。这些能力其实是三种东西组合出来的，搞懂它们，你才会知道怎么扩展自己的 AI。

把 CLI AI 想象成一个新搬进你家的厨子：
- Tools = 厨房里已有的锅碗瓢盆（他能用的能力）
- MCP = 你给厨房加新设备的接口（比如可以接烤箱、空气炸锅）
- Skills = 你交给他的食谱本（不增加工具，但教他"做某道菜该怎么做"）

1. Tools中文"工具"。AI 在回答时不只是"输出文字"，还可以"调用一个函数"去做事——查天气、读文件、跑命令。每次 AI 决定调用工具，就叫一次 "tool call"。 —— AI 能做的"具体动作"

Claude Code 启动后内置了一组工具，你可以直接看到：

工具名作用
Read读你电脑里的文件
Write / Edit新建或修改文件
Bash跑终端命令（ls、ping、git 等都靠它）
Grep / Glob在文件里搜内容、找文件
WebFetch抓取网页内容

每次它"决定动手"，本质就是调用了这些工具中的一个。每次调用默认都会请示你"允许吗？"。

2. MCP全称 Model Context Protocol（模型上下文协议），Anthropic 在 2024 年底推出的开放标准。简单说就是"AI 工具的 USB 接口"——任何人写一个 MCP server，所有支持 MCP 的 AI 都能即插即用。 —— 给 AI 接上"外挂能力"

内置工具不够用？MCP 就是为此而生：它定义了一套"插头标准"，让任何人都能开发一个"AI 外设"，AI 接上后立刻多一组新工具。

常见的 MCP server 比如：

  • filesystem：给 AI 限定可访问的本地目录
  • github：让 AI 能查 PR、Issue、读仓库代码
  • sqlite / postgres：让 AI 能查你的数据库
  • puppeteer：让 AI 能控制浏览器
  • fetch：让 AI 能稳定抓取 URL 内容
Tools 是出厂自带的功能；MCP 是"AI 的应用商店"——你需要什么能力，就装哪个 MCP server。

3. SkillsClaude Code 的"做事食谱"。一段 markdown 文件，告诉 AI："遇到这类任务时，按下面这个流程办，注意这些坑。" 与提示词的区别是：Skills 是按需自动加载的，不会一直占用上下文。 —— 教 AI "怎么做某件事"

Skills 不增加新工具，而是固化"做某类事的方法"。它本质上就是一个 markdown 文件，描述："当用户想做 X 时，按以下步骤、用以下风格、注意以下坑。"

Skills 和提示词的区别：

  • 提示词每次对话都要重写或粘贴
  • Skills 一次写好，AI 自动识别合适场景调用

三者关系一张图

你说："帮我整理桌面上的照片"
↓
Skill"photo-organize"被触发（告诉 AI 该怎么做）
↓
AI 调用 Tool Bash 列文件，再调用 MCP filesystem 安全地移动文件
↓
完成

四、轻量实战 1：5 分钟给 Claude Code 写一个 Skill（零依赖）

这个实战不需要装任何东西，只要你已经装好 Claude Code，新建一个 markdown 文件就行。

步骤

1创建 skill 目录 
$ mkdir -p ~/.claude/skills/weekly-report
2新建 SKILL.md，用任何文本编辑器（VSCode、记事本都行）写入： 
---
name: weekly-report
description: 把零散的工作笔记整理成结构化周报。当用户提到"周报""周总结""work review"时触发。
---

# 周报助手

当用户需要写周报时，按以下流程：

1. 先问用户："本周做了哪些事？可以贴一段笔记，也可以一条一条说。"
2. 把内容自动归类为三组：
   - 🎯 重要交付（推动核心目标的事）
   - ⚙️ 日常运维（必要但不显眼的事）
   - 📚 学习成长（看的书、学的工具、踩的坑）
3. 输出 markdown 格式：
   ```
   ## 本周完成
   ## 进行中
   ## 下周计划
   ## 风险/需协助
   ```
4. 末尾给出 1 句"亮点提炼"用作汇报开场白。

提醒：宁缺勿增，不要替用户编没发生的事。
3启动 Claude Code，验证 skill 被加载 
$ claude
> /skills
# 应该能在列表里看到 weekly-report
4触发它 
> 帮我写一份周报
# Claude Code 会自动加载 weekly-report skill，按你定义的流程提问、整理、输出
Skill 文件的第一行 description 非常重要——AI 是根据它判断"现在这个对话要不要加载这个 skill"。description 写得越精准，命中率越高。

更多可以照着写的轻量 Skills

  • 日记整理：把语音转写的杂乱文字整理成结构化日记
  • 中英对照翻译：固定按你偏好的术语和语气翻译
  • 读书笔记：从一本书的章节笔记提炼核心观点 + 行动清单
  • 会议纪要：从会议录音文字稿提取议题、决策、待办

五、轻量实战 2：启用一个 MCP（零安装）

很多 MCP server 都用 npx 启动—— npx 是 Node.js 自带的"用时下载、用完不留"运行方式，不需要全局安装任何服务。我们用最简单的 filesystem MCP 演示。

它能做什么？

给 AI 一个"安全沙盒"——AI 只能在你指定的目录里读写文件，碰不到目录外的东西。比让 AI 满硬盘乱跑安全得多。

步骤

1找到 Claude Code 的配置文件（首次没有就新建） 
$ mkdir -p ~/.config/claude-code
$ open ~/.claude.json  # macOS

不同版本路径略有差异，claude config 命令可以告诉你当前用的是哪个。

2添加 mcpServers 段（如果文件已有别的内容，把 mcpServers 字段合并进去） 
{
  "mcpServers": {
    "fs": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/你的用户名/Desktop"
      ]
    }
  }
}

路径换成你想让 AI 能访问的那个目录（这里限定在桌面，比较安全）。

3重启 Claude Code，验证 
$ claude
> /mcp
# 看到 "fs" 已连接，并列出了它提供的工具（list_directory、read_file、write_file...）
4试一下 
> 用 fs 工具看一下我桌面上有哪些文件，给我列一个清单。

注意：第一次 npx 会临时下载 server 包（几兆，几秒），之后会缓存。

想看更多 MCP？官方仓库收录了几十个开箱即用的 server：github.com/modelcontextprotocol/servers，按需把 args 一改就能用。

Skill vs MCP 该选哪个？

你想…用
给 AI 固化"做某类事的方法 / 风格"Skill
给 AI 接上一个新外部数据源或新动作MCP
把流程写死给所有同事共享Skill（一个 markdown 拷贝即可）
调用真实 API / 数据库 / 浏览器MCP

六、家用场景：用 CLI AI 优化你的电脑和网络

场景 1：诊断 Wi-Fi 慢、网页打不开入门

对它说： 
> 我电脑最近上网很慢，帮我系统地检查一下网络情况：测速、看 DNS、看路由跳数、看 wifi 信号强度，最后给我一份诊断报告和优化建议。
它会自动跑：
  • ping 223.5.5.5 / ping www.baidu.com（看丢包/延迟）
  • traceroute（看从你这到目标网站走了几跳，哪一跳卡）
  • scutil --dns（macOS 看当前 DNS 配置）
  • networksetup -getairportnetwork en0（看连的是哪个 wifi）
  • 调用 speedtest-cli（如已安装）测速
  • 读 system_profiler SPAirPortDataType 看信号强度、信噪比
它给你的报告大致：
诊断结果：到百度延迟 12ms 正常，但到 GitHub 600ms 且第 5 跳丢包 30%——典型国际线路拥塞。DNS 用的是路由器默认的运营商 DNS，可换 223.5.5.5（阿里） 或 1.1.1.1（Cloudflare） 提升解析速度。WiFi 信号强度 -72dBm 偏弱，建议靠近路由器或换 5GHz 频段。

场景 2：改 DNS / Hosts，立刻见效入门

> 把我 Mac 的 DNS 改成阿里和 Cloudflare（备用 8.8.8.8），改完后帮我测一下解析速度有没有提升。

它会用 networksetup -setdnsservers Wi-Fi 223.5.5.5 1.1.1.1 8.8.8.8，然后跑 dig 对比前后耗时。

场景 3：SSH 登录软路由（OpenWrt / iKuai / RouterOS）优化配置中等

这是 CLI AI 真正的"杀手锏"场景之一——你不用记一堆 Linux 命令、不用查 OpenWrt 文档。

前提：你有一台软路由（如 NanoPi、N100 小主机刷 OpenWrt / iStoreOS），知道 IP 和 SSH 密码。

对它说： 
> 帮我 SSH 到 192.168.1.1（用户 root），先检查这台 OpenWrt 的：
1. CPU、内存、温度
2. 网络接口（WAN/LAN）状态
3. 现在开启了哪些服务
4. 无线信道是否冲突
然后给我一份"可以这样优化"的清单，但每条改动改之前都要先问我确认。
它会自动做：
  • ssh root@192.168.1.1 登录
  • 跑 top、free -m、cat /sys/class/thermal/thermal_zone0/temp
  • ifstatus wan、ifstatus lan
  • /etc/init.d 下看启用的服务
  • iw dev wlan0 scan | grep -E 'freq|signal' 看周围信道占用
典型优化建议：
  • 切换 2.4G 到 1/6/11 中冲突最少的信道
  • 启用 Software / Hardware Flow Offloading 提升转发性能
  • 开启 Cake QoS 解决游戏卡顿
  • 启用 DNS over HTTPS（DoH）
  • 开启 IPv6 PD（如运营商支持）
路由器是家里所有设备的"大门"，改动前请先在路由器后台备份配置。一旦改坏导致断网，可以恢复。如果你完全是新手，建议每条变更都让 AI 先解释一遍再执行。

场景 4：清理 Mac / Windows 上的垃圾文件入门

> 扫描我用户目录下大于 500MB 的文件，按大小排序列出来，并标注哪些是缓存/可以安全删的，但删除前一定要让我确认每一项。

它会用 find ~ -type f -size +500M，并理解 ~/Library/Caches、node_modules、Docker 镜像等是可清理的。

场景 5：批量处理照片/视频入门

> ~/Pictures/旅游 下所有 jpg 帮我：①统一缩到长边 1920 ②文件名改成"日期-序号" ③转成 webp 节省空间，保留原图到 /原图 子目录。

它会用 ffmpeg 或 sips（macOS 自带）一条命令搞定，比手动一张张处理快百倍。

场景 6：搭建家庭 NAS / 内网服务中等

> SSH 到 192.168.1.5（家里的群晖 / 树莓派），帮我用 docker compose 部署一个 Vaultwarden（密码管理器），配好 HTTPS 自签证书，端口 8443，部署完告诉我从手机怎么访问。

场景 7：定时任务中等

> 写一个 macOS launchd 任务，每周一早上 9 点自动把 ~/Documents 备份到我挂载的 NAS（//nas.local/backup），并把备份日志发到我邮箱。

场景 8：分析路由器日志找异常较难

> 下载我路由器最近一周的 syslog，分析有没有异常登录尝试、ARP 欺骗、可疑外联，把可疑 IP 列出来并查一下归属。

七、用 CLI AI 的 5 条安全守则

1. 永远在"专门的工作目录"启动它。不要直接在桌面或 / 根目录用，免得它操作范围太大。

2. 危险命令要看清楚。遇到 rm -rf、sudo、chmod 777、curl ... | sh 这类，先让它解释再放行。

3. 重要文件先备份。路由器改配置前导出备份；改电脑系统设置前记下原值。

4. 不要把密码、密钥贴进对话。用环境变量或 SSH key，不要明文。

5. 学会随时打断它。看到方向不对，直接 Ctrl+C 停下，再用自然语言调整方向，不要硬看它跑完。
