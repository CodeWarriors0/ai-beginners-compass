npm（Node.js）安装教程课外 · 工具准备

为什么需要它 · 怎么装 · 装完怎么验证 · 常见踩坑

你在 课外-本地AI命令行工具 里看到的 npm install -g @anthropic-ai/claude-code 这类命令，就是用 npm 来安装 CLI AI 工具。本节带你把 npm 装好，让那一行命令能跑得起来。

一、几个名词先搞清楚

名词是什么
Node.js一个能让 JavaScript 跑在电脑（而不是浏览器）里的运行环境。很多命令行工具都用 JavaScript 写的，所以要先装 Node.js 才能跑。 JavaScript 的"运行环境"，许多命令行工具用它写成
npmNode Package Manager，Node.js 自带的包管理器。装 Node.js 就会自动一起装上 npm，不需要单独装。 Node.js 自带的"包管家"，用来一行命令装/卸软件
npm install -g xxx "全局安装 xxx"——装完后系统任何地方都能用
npx xxx "临时跑一下 xxx"——不安装，用完即走
Node.js 是一台"洗衣机"，npm 是这台洗衣机自带的"洗衣液柜台"。装好洗衣机，柜台自动就有了。所以本教程其实就是装 Node.js——npm 自动到位。

二、按你的电脑系统选一种装法

macOS Windows Linux

macOS 推荐方式：官网下载安装包（最省心）

1打开官网：访问 https://nodejs.org
2下载 LTS 版本：首页大按钮，选 LTS（长期支持版），不要选 Current。下载到 .pkg 文件。
3双击安装：一路点"继续/同意/安装"，需要输入电脑开机密码。
4验证：打开"终端"（Spotlight 搜 Terminal），分别输入下面两条：
$ node -v v22.10.0 $ npm -v 10.9.0

能输出版本号就说明装好了（数字可能不一样，别在意）。

熟悉命令行的同学也可以用 Homebrew：brew install node。但首次安装建议走官网包，不易出错。

Windows 推荐方式：官网下载安装包

1打开官网：访问 https://nodejs.org
2下载 LTS：下载 .msi 安装文件。
3双击安装：勾选"Add to PATH"（默认就勾着，不要取消），一路 Next。
4验证：开始菜单搜 PowerShell 打开，输入：
PS> node -v v22.10.0 PS> npm -v 10.9.0
Windows 上如果输入 node -v 提示"不是内部或外部命令"——通常是安装时没勾"Add to PATH"，把 Node.js 卸了重装并保留默认选项即可。

Linux 推荐方式：用 nvm（更灵活）

# 1. 装 nvm（Node 版本管理器） $ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash # 2. 关闭重开终端，然后装最新 LTS $ nvm install --lts # 3. 验证 $ node -v $ npm -v

nvm 的好处：以后想换 Node 版本，一行命令就行（nvm install 20 等）。

三、用 npm 安装你的第一个工具

Node.js 装好后，回到 课外-本地AI命令行工具 表格里，复制对应命令到终端里跑。例如安装 Claude Code：

$ npm install -g @anthropic-ai/claude-code

解读这一行：

  • npm —— 调用 npm
  • install —— 安装
  • -g —— 全局安装（装完所有目录都能用）
  • @anthropic-ai/claude-code —— 要装的包名

跑完后，终端会显示一行 added N packages in Xs。然后直接输入：

$ claude # 进入 Claude Code 交互界面，按它的提示登录即可

四、4 个常见踩坑

❌ 坑 1：EACCES: permission denied（Mac / Linux）
✅ 别用 sudo npm install（不安全）。改用 nvm 装的 Node，或者给 npm 改默认目录到家目录。最简单解决：装 nvm。

❌ 坑 2：下载特别慢 / 超时
✅ 换国内镜像： $ npm config set registry https://registry.npmmirror.com
以后想换回官方：npm config set registry https://registry.npmjs.org

❌ 坑 3：装完命令找不到（如 claude: command not found）
✅ 通常是 npm 全局目录没在 PATH 里。先看路径：npm config get prefix，再把对应的 bin 子目录加到 PATH。或者最干脆——用 nvm 重装 Node，问题会自动消失。

❌ 坑 4：升级 npm 自身
✅ 直接：npm install -g npm。Node.js 也可以更新（重新下安装包覆盖即可，或 nvm install --lts --reinstall-packages-from=current）。

五、卸载与清理

装多了想清场？

# 卸载某个全局包 $ npm uninstall -g @anthropic-ai/claude-code # 列出所有全局安装的包 $ npm list -g --depth=0

整个 Node.js 卸载：

  • macOS（官网包安装的）：可用 官方卸载脚本，或直接装新版覆盖。
  • Windows：控制面板 → 程序和功能 → 找到 Node.js → 卸载。
  • nvm：nvm uninstall <版本号>。
